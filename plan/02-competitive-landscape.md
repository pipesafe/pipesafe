# 02 — Competitive Landscape

PipeSafe sits at the intersection of two markets that have never been served by one tool: the **application data layer** (Mongoose, Prisma — where developers read and write MongoDB from application code) and the **analytics/transform layer** (dbt, Fivetran/Airbyte/dlt, Dagster/Airflow/Prefect, and emerging control planes like Orchestra — where teams model, materialize, and orchestrate derived data). Both markets validate PipeSafe's core thesis from opposite directions: Mongoose explicitly ceded typed aggregation (`aggregate<R = any>`), and dbt Labs spent an acquisition plus a Rust rewrite (Fusion) to buy static comprehension of its transformation language — a property PipeSafe's TypeScript inference provides natively. The one direct, contested threat is **Prisma Next**: an Apache-2.0 TypeScript rewrite shipping a typed MongoDB aggregation builder in partnership with MongoDB Inc., currently Early Access. This document maps the field; roadmaps responding to it are in `03-orm-roadmap.md`, `04-transform-roadmap.md`, and `05-orchestration-and-el-roadmap.md`.

## 1. Market map

PipeSafe straddles two axes: _where the code runs_ (application runtime vs. batch/pipeline runtime) and _what is typed_ (nothing / hand-asserted / codegen'd / inferred). No incumbent covers both quadrants with compile-time types.

| Layer                    | Incumbents                                               | Typing model                                                      | MongoDB aggregation story                                                                        |
| ------------------------ | -------------------------------------------------------- | ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| **ORM / app data layer** | Mongoose                                                 | Hand-written 10.5k-line `types/` chasing a 30k-line JS runtime    | Explicitly untyped: `aggregate<R = any>`, "Mongoose does not cast pipeline stages"               |
|                          | Prisma 7                                                 | Codegen from `schema.prisma` (~50-package toolchain)              | `aggregateRaw(pipeline?: InputJsonObject[]): PrismaPromise<JsonObject>`; MongoDB pinned to v6.19 |
|                          | **Prisma Next** (EA)                                     | Contract-first: emitted `contract.json` + `contract.d.ts`         | Typed pipeline builder, 18+ stages — the direct threat (§2)                                      |
| **Transform (T of ELT)** | dbt Core / Fusion                                        | Stringly-typed Jinja+SQL; Fusion retrofits a Rust static analyzer | N/A — warehouse-only                                                                             |
| **EL / ingestion**       | Fivetran, Airbyte, Singer, dlt                           | Runtime schemas (protobuf enums, JSON Schema, Python inference)   | MongoDB as _source_ only; no serious MongoDB _destination_ exists                                |
| **Orchestration**        | Dagster, Airflow, Prefect, Temporal, Inngest/Trigger.dev | Python (untyped graphs) or TS (dynamic step discovery)            | None model data shapes at all                                                                    |
| **Control plane**        | Orchestra                                                | Declarative YAML over other tools' jobs                           | Consumes artifacts (dbt `manifest.json`); owns skip-state above the runner                       |
| **PipeSafe**             | `@pipesafe/core` + `@pipesafe/manifold`                  | Inferred from plain TS types — zero codegen, zero runtime engine  | Native: stage-by-stage schema rewriting, union narrowing, typed `$lookup`, DAG materialization   |

**A third competitor family sits off this table: the BaaS platforms.** Supabase ($10.5B, ~$170M ARR, and — decisively — the default backend of the AI-builder channel, with AI agents deploying the majority of its new databases) and Convex (the typed-TS, document-model cousin, at ~1/200th the valuation — channel beats DX) compete for the same developers one layer up: one SDK for data + auth + realtime + storage, secured by row-level security. MongoDB built and killed its own entry in this family — Atlas App Services went dark September 30, 2025 — and has since delegated the layer to partners (Appwrite for the platform, Prisma for the client), leaving the _typed data plane_ (typed queries, typed live subscriptions, typed policy enforcement) unowned. PipeSafe does not compete with this family on platform surface; it competes on the layer the family is weakest at — supabase-js's filter builder admits untyped fallbacks and its query ceiling forces SQL escape hatches, and Supabase Realtime's WAL-polling fan-out is its most complained-about pillar. The full assessment and the resulting wedges (EPIC-K/L/M) are in `08-supabase-baas-assessment.md`.

The strategic pattern: every layer is independently converging on PipeSafe's position. Prisma unwound its Rust engine (v7) and then minimized codegen and embraced schema-in-TypeScript (Next). dbt is bolting a type system onto Jinja (`crates/minijinja-typecheck-builtins`). Airbyte spent three years converging on Fivetran's typed-on-arrival model. dlt proved developers want EL as typed library code. Dagster's asset model is manifold's `Model` almost term-for-term.

## 2. ORM competitors

**Mongoose (9.x)** is runtime-first: a JavaScript casting/validation/hooks engine with types bolted on afterwards. Its strengths are real — kareem-based middleware powering the entire plugin ecosystem, populate's batching discipline (`$in` splitting under the 16MB BSON limit), best-in-class transaction ergonomics with `AsyncLocalStorage` session propagation. Its weaknesses are structural: `Schema` carries eleven type parameters; filter typing goes one level deep (`WithLevel1NestedPaths`, the name is the confession); populate typing is user-asserted (`populate<{author: Author}>('author')`); and aggregation is abandoned by explicit design — `Model.aggregate<R = any>` types the result as whatever the user hopes, the pipeline against a schema-agnostic `PipelineStage` union, and v9 banned update pipelines by default because they're uncastable. Mongoose has no equivalent of `PipeSafeError`-curated diagnostics; inference failures dump internal alias soup. **The moment a Mongoose app needs `$group`/`$lookup`/`$project`, all safety evaporates.**

**Prisma 7** is codegen-centric (`schema.prisma` → DMMF → generated client), and its flagship Rust-free architecture — WASM query compiler plus TS runtime, 3.4x faster, 90% smaller — **excludes MongoDB entirely**. Mongo users are pinned to v6.19 on the legacy Rust binary, with no `prisma migrate` (only index sync via `db push` — the success message is literally branched per provider in `DbPush.ts:258`), a replica-set requirement imposed by nested-write transactions, null-vs-missing semantics collapsed, and everything past the fixed `_count/_avg/_sum/_min/_max` menu falling into untyped `aggregateRaw` that additionally demands Extended JSON (`{ $oid: ... }`) — a dialect the rest of the client never exposes. Prisma's commercial gravity (Prisma Postgres, operations-based billing) is SQL-shaped; MongoDB is visibly not the priority.

**Prisma Next is the direct threat.** The `prisma/prisma-next` repo (v0.14.0, Apache-2.0) is a ground-up TS rewrite — no Rust, no generated client, a versioned `contract.json` with sha256 drift detection — and MongoDB support arrives _exclusively_ through it, built "actively working with the Mongo team." Its `@prisma-next/mongo-query-builder` is a near-clone of PipeSafe's surface: 18+ stages including `lookup`, `unionWith`, `facet`, `setWindowFields`, `$out`/`$merge` terminals; per-stage result-shape tracking via a phantom `Row` type; typed `$lookup` with foreign-field checking; typed update operators with marker-gated terminals; a contract-free inference mode (`AggregateChain<Shape>`); discriminated-union polymorphism; and typed `Migration<Start, End>` data migrations powered by pipelines — the exact design space manifold occupies. The honest differences, from their own docs: it is EA quality with documented type holes (`lookup` root-name typos accepted at compile time, TML-2400, because `Contract.roots` is `Record<string, string>`); the expression vocabulary is thin (`acc.sum/count/max` vs. PipeSafe's `$dateToString`/`$ifNull`/full operator set); polymorphism must be _declared_ via contract discriminators rather than _inferred_ from `$match` predicates; the contract emit step reintroduces codegen friction (their own example README warns of types silently widening to `any` under pnpm strict hoisting); and the syntax is a callback DSL (`f.status.eq('completed')`, `acc.sum(f.amount)`) that compiles to Mongo, versus PipeSafe's literal aggregation syntax (`{ status: 'completed' }`, `{ $sum: '$amount' }`) that copy-pastes to and from `mongosh`, docs, and LLM training data. **The window is real and bounded**: Prisma Next's roadmap GAs Postgres first (becoming Prisma 8, mid-2026); MongoDB is explicitly a framework-validation POC with no committed GA date — plausibly 2027. PipeSafe is shipping today. Mindshare for "the typed Mongo pipeline tool" gets decided in that gap.

## 3. Transform / ETL competitors

**dbt** defines the transform category and its vocabulary (models, `ref()`, incremental, snapshots, seeds, exposures). Its incremental materializations — `is_incremental()` delta filters plus merge/delete+insert/microbatch strategies — and its state artifacts (`manifest.json`, `state:modified` diffing against _unrendered_ configs, `--defer` remapping unchanged upstreams to prod) are the workflows manifold must match (`04-transform-roadmap.md`). Its weakness is the wedge: everything is stringly typed. `ref('stg_orders')` and every column name are strings; renames fail hours into a warehouse run; debugging is archaeology through `target/compiled/`. dbt Labs' answer, **Fusion** (now dbt Core v2.0, Apache-2.0), is a Rust rewrite on forked DataFusion with per-dialect ANTLR lexers, column-level lineage (`CllEdge`), grain inference, and — most tellingly — a _type checker for Jinja_. dbt spent an acquisition and a multi-year rewrite to obtain static comprehension of its transformation language. PipeSafe's pipelines are typed data structures; the same comprehension is native and stronger (TS types are ground truth, not best-effort dialect parsing). Also instructive: dbt's raw-text `same_body` comparison is its weakest state primitive — manifold's pipelines-as-JSON get semantic hashing for free.

**Fivetran** sells the absence of engineering: ~700 connectors, cursor-checkpointed syncs, an SDK protocol (`connector_sdk.proto`) whose op vocabulary — `UPSERT | UPDATE | DELETE | TRUNCATE` plus opaque checkpointed `state_json` — maps one-to-one onto Mongo bulk ops. Its pain is **MAR pricing** (per-connection since March 2025; bill-doubling reports are the loudest migration driver in the category), and its consolidation with **dbt Labs** (merger closed June 2026) means EL+T is now one vendor — validating the integrated-stack thesis while leaving MongoDB out entirely. **Airbyte** (ELv2 core) formalized Singer's stdout protocol; its MongoDB _destination_ is archived. **dlt** is PipeSafe's closest analogue: an Apache-2.0 library, not a platform — typed-at-runtime connectors as Python generators, incremental cursors stored in the destination, schema contracts (`evolve`/`freeze`/`discard`). dlt proves the "EL as library code in your own runtime" market; it just can't make the schema a compile-time object. A TypeScript equivalent can.

**Orchestrators.** Dagster's software-defined assets are manifold's `Model` almost exactly (`@asset` ↔ `Model`, `deps` ↔ `from`+lookup auto-discovery, `Definitions` ↔ `Project`), and its architecture lesson is uniform across Dagster/Prefect/Temporal/Inngest: _everyone persists an event log keyed by stable identifiers; manifold today keeps orchestration state only in process memory_. Dagster's `code_version` staleness, partitions+backfills, and the dagster-dbt pattern (parse dbt's manifest, stream per-node events from one CLI invocation) define the integration seam manifold should expose. **Orchestra** is not a competitor but a warning: a closed SaaS control plane that started by _triggering_ dbt and then absorbed the runner beneath it — a managed dbt Core runner plus State-Aware Orchestration, a workspace state store that skips models whose code and upstream freshness are unchanged (claimed 30–75% fewer builds). The lesson: **value accrues to whoever owns the skip-state**. If manifold ships only a runner, the state layer — and its margin — gets built above it; if manifold persists materialization state, pipeline hashes, and source freshness natively, it keeps that layer inside the product (`05-orchestration-and-el-roadmap.md`, `06-architecture-packaging-licensing.md`).

## 5. Strategic implications

The wedge statements, one per front:

- **vs. Mongoose:** _PipeSafe is what `Model.aggregate()` should have been._ Their own artifacts concede it. The ORM growth path (`03-orm-roadmap.md`) is to surround that core with the daily 20% — full-depth typed filters vs. their level-1, type-visible hooks, transactions — while skipping the 11k lines of casting machinery that exist to compensate for missing compile-time guarantees.
- **vs. Prisma:** _Everything past `findMany`, typed._ Against shipped Prisma that's simply true (`aggregateRaw: JsonObject`); against Prisma Next, differentiate on typing _depth_ (inferred narrowing vs. declared discriminators; literal-keyed schemas vs. typo-accepting roots), literal syntax, and zero emit — and build the head-to-head against their `mongo-blog-leaderboard` example now, inside the window before Prisma 8's Mongo GA.
- **vs. dbt:** _dbt for MongoDB, without Jinja._ Renames are compile errors across the whole DAG; macros are typed functions; packages are npm. Fusion is the strongest third-party validation of the thesis in existence — and dbt's vocabulary (incremental, snapshot, seed, exposure) should be adopted wholesale so migration is translation, not education.
- **vs. orchestrators and Orchestra:** _The typed, TS-native asset layer that any scheduler can host and any control plane can read_ — manifest + CLI + structured events, the dbt seam — while keeping the skip-state (pipeline hashes, freshness, event log in MongoDB itself) in the product.
- **vs. Supabase and the BaaS family:** _The typed realtime + RLS layer Atlas lost._ Not a platform play — agent-channel assets now (EPIC-K), typed live queries over change streams against Supabase's weakest pillar (EPIC-L), and out-of-the-box RLS as the Supabase → Atlas + PipeSafe migration unlock (EPIC-M). Detail and counter-arguments in `08-supabase-baas-assessment.md`.

**Uncontested lanes:** MongoDB as an EL _destination_ (Airbyte's is archived, Fivetran has none — "first serious typed EL into MongoDB" has no incumbent), change-stream-driven continuous materialization, and typed unit-test fixtures checked against `TInput`/`TOutput` at compile time — something dbt structurally cannot do. **The contested lane** is exactly one: Prisma Next's typed Mongo builder. It is EA, syntax-forked, and shallower-typed today, but Apache-licensed, MongoDB-partnered, and shipping weekly. Every quarter of the roadmap should be scheduled against that clock.
