"@pipesafe/core": minor

Add connection and query option support: `pipesafe.connect()` now accepts driver `MongoClientOptions` (custom timeouts, pool sizing, etc.), `pipesafe.db()` / `Database` / `Collection` accept driver `DbOptions` / `CollectionOptions` (read/write concern, read preference), and `Pipeline.execute()` accepts `AggregateOptions` — including `session`, enabling aggregations inside transactions. Each layer configures its own options where it is created (`pipesafe.connect()` → client, `pipesafe.db()` / `Database` → database, `Database.collection()` / `Collection` → collection) and they flow down to execution — `Pipeline.execute()` takes only the aggregation's own `AggregateOptions` and cannot override a parent layer's configuration. The `Source` interface gains `getOutputDbOptions()` / `getOutputCollectionOptions()` accessors (implemented by `Collection` and `Model`, the two intended implementers) so consumers reading a source through the interface can honor its configured options.
